Search Syntax Guide & Examples

Construct advanced queries by combining multiple rules using AND / OR logic. This search engine seamlessly queries across Assets, Targets, Ports, Web Servers, and Vulnerabilities simultaneously.

Matching Operators:
  • Matches Pattern (*): Uses SQL wildcards. Use * to match any characters. (Highly efficient)
  • Matches Regex: Uses native Go regular expressions for precise filtering. (Evaluated post-fetch)
  • Equals / Contains: Standard exact or partial text matching.
  • Greater/Less Than: Useful for numeric fields like Ports or Status Codes.
